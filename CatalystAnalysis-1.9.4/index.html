  • US1234128 Show/Hide details : #IT - WERS refresh data bump
    11/05/2019 Released
    Processes program BOM files and identifies WERS suffix bump and validates effectivity In/Out dates for proper consistencies

    Description
    As a member of FEDEBOM IT
    I want to Understand the data quality issues regarding WERS refresh data bump So that I can create a toolset which can automatically identify data quality issues

    Acceptance Criteria
    Given FEDEBOM program data
    When WERS refresh causes a data bump (suffix code increment) issue
    Then logically identify proper usage of such data w/respect too effectivity In/Out values of each component

    Method to find soft matched suffix values based on below rules.

    1. If the suffix is 1 character in length, just match on the prefix and base (e.g. A soft matches B).
    2. If 3rd position is "W" followed by blanks, soft match on the 2nd position of the suffix (e.g. AAW soft matches ABW).
    3. If 4th position is "W" followed by blanks, soft match on the 3rd position of the suffix (e.g. AAAW soft matches AABW).
    4. For suffixes starting with at least 1 alpha character and ending in a string of all numbers (up to 6 numbers), soft match on the alpha portion less one character ignoring the numeric portion. The length of the suffix and the length and value of the numeric portion of the suffix must match. For example, AA30 advances to AB30, but AB40, AB31, AA301 are not soft matches.
    5. For all other non-blank suffixes, soft match on the last position of the suffix (e.g. AA soft matches AB, A1A soft matches A1B, AAAA soft matches AAAB, etc.)
    6. For blank suffixes, soft match on prefix and base only (exact match only - no predecessor)
    7. DE59185: B1 is not soft match with BD. If we remove all digits from a suffix and we are left with only one character, then there is no soft match.
